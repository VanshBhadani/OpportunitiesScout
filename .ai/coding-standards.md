# Coding Standards

- Keep changes scoped to the requested mission or worker packet.
- Prefer existing project patterns over new abstractions.
- Use Markdown for orchestration policy and memory.
- Keep prompts factual, imperative, and easy for agents to follow.
- Avoid hardcoding models, skills, validators, or workflows in multiple files.
- Record fallback behavior whenever a preferred tool or model is unavailable.
- Do not install dependencies or browser binaries without explicit approval.
- Do not commit without explicit approval.
